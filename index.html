Case 01  ·  Wipro | Meta Account Current

An AI delivery intelligence system that gave 450+ engineers real-time visibility — and gave their managers AI superpowers.

Because at Meta's scale, "I'll check manually" is not how you run a delivery operation.

↓
Hatch / Meta Nest Stack Meta's proprietary engineering delivery platform
← Meta's proprietary infrastructure
↓
Hatch Dashboard real-time delivery health  ·  KPI tracking
Metamate + Claude Skills AI insights  ·  productivity layer  ·  team performance
↓
Delivery Leaders + Client Reports real-time  ·  shareable  ·  cadenced  ·  automated

engineering activity flows in, delivery intelligence flows out — and straight to the client

The Problem

Engineering delivery for the Meta team was invisible. Tracking diffs landed, code review cycles, and task throughput required manual effort across Meta's proprietary Hatch/Nest systems. Service delivery had no real-time health signal — and AI infusion was ad-hoc and person-dependent, not a practice.

Approach

Mapped Meta's Nest-based engineering infrastructure and connected it to a custom dashboard layer. Added Metamate — Meta's enterprise LLM built on Claude — as the AI reasoning layer. Designed Claude-based skills for delivery managers to track performance and drive team output without manual effort. Built and led a 25-member dedicated AI delivery team within the Meta ecosystem.

Solution

The Hatch Dashboard connects to Meta's Nest stack and aggregates diffs landed, review comments, and task throughput into a live delivery health view — shareable directly with the Meta client. The Claude skills framework gives delivery managers structured AI-driven insights. 10+ agents delivered across agentic workflows embedded into ongoing delivery operations.

Outcome

Eliminated manual tracking entirely. Delivery leaders now have a real-time, client-shareable engineering health dashboard. Skills framework running across the service delivery practice — measurable ~20% reduction in manual effort, improved customer experience across select Meta workflows, and AI infusion at scale.

Project 01  ·  BazaarGaram Personal Project

An options chain analyser for NIFTY & BANKNIFTY that surfaces strategy recommendations — without placing a single order.

Because good options trading starts with reading the market structure, not reacting to it.

NIFTY / BANKNIFTY Angel One SmartAPI  ·  live options chain
↓
Spot Price Engine 3-fallback: SmartAPI → Scrip Search → Put-Call Parity
↓
Options Chain Fetcher 8 strikes ATM ±  ·  CE/PE LTP  ·  OI  ·  Volume
↓
Analysis Engine PCR  ·  Max Pain  ·  OI Support/Resistance  ·  IV Skew
↓
Strategy Recommender straddle  ·  bull spread  ·  iron condor  ·  strangle
↓
Recommendation Output entry  ·  stop loss  ·  target  ·  payoff profile

market structure first, strategy second — never the other way around

The Problem

Reading options market structure manually — PCR, Max Pain, OI levels, IV skew across strikes — takes 30+ minutes before a session. Most traders skip it and trade on gut feel, reacting to price rather than understanding where the market is positioned.

Approach

Pure quantitative analysis engine — no LLM needed here, the math is deterministic. Fetch live options chain from Angel One, run four market structure signals in parallel, then map to the most appropriate strategy payoff profile. Three-fallback spot price logic handles Angel One API inconsistencies gracefully.

Solution

Single API endpoint — pass underlying + expiry type + strategy → get full chain analysis + recommendation in seconds. Covers 6 strategy types: simple buy, bull call spread, bear put spread, straddle, strangle, iron condor. No orders placed — analysis and conviction, user decides.

NIFTY+ Banknifty

Project 03  ·  Antigravity — Wealth PMS Personal Project

An autonomous portfolio management agent that monitors Indian equity holdings, reasons over live market data, and proposes trades — governed by strict risk guardrails.

Because portfolio decisions shouldn't depend on whether you're watching the screen at 10 AM.

Sharekhan Broker API holdings  ·  available cash  ·  margin
↓
Perception Node portfolio state  ·  total valuation
↓
Reasoning Node — GPT-4 yfinance  ·  RSI / MACD / Bollinger  ·  Tavily news  ·  temp=0
↓
Guardrails Validator market hours  ·  10% concentration  ·  2% drawdown
↓
Execution Node order log  ·  duplicate prevention  ·  broker placement

reasoning is deterministic, risk management is non-negotiable

The Problem

Active portfolio management requires constant monitoring — checking holdings, running technical analysis, reading news sentiment, and making disciplined trade decisions without letting emotion in. It's not a part-time job.

Approach

LangGraph state machine with three nodes: Perception (portfolio state from Sharekhan), Reasoning (GPT-4 at temperature=0 — deterministic — with yfinance, RSI/MACD/Bollinger, and Tavily news as tools), and Execution (risk-validated broker placement). Guardrails configurable via environment variables.

Solution

On-demand agent that fetches live holdings, reasons over technical indicators and news sentiment, validates against risk guardrails, and logs execution. Idempotent — won't execute twice in the same trading session. Current build uses mock broker for safety; live Sharekhan API endpoints are wired and ready.

GPT-4Reasoning Engine
3Risk Guardrails
"deterministic — never double-executed"
